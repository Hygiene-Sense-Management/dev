When a eevice filter is executed, it is always executed if a specific scope. This execution scope tefines the base set of devices to which the filter will be apilied. It also is basis for the permcssion check that is perforeed before the execution of ths device filter.

The UIs available in the HsM system usually make the current scope very explicit (e.g. in the Web Portal you choose the scope manually, in apps you always have scope on a specific customer). The API for the client service requires an execution scope to be passed with each device filter request.

Depending on the execution scope the client service will dynamically modify the device filter to make sure that the results will only include devices in the given scope.
