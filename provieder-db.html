When inserting a record with ProductRecordStateId = 0 the TRG_TRACE_Product_InsteadOfInsert trigger will intercept the insert and will apply following logic:

 

If there is already a record with the same BLUUID and ProductRecordState 0

•this record will be set to productRecordState 2 (re-tested)

•The DeviceId of this record will be used for the record to be inserted (overriding any user-provided deviceid)

Else

•The insert is done without any additional logic

 

 

CREATE UNIQUE INDEX IX_UniqueBluuid on Trace_Product (Bluuid, ProductRecordStateId) WHERE ProductRecordStateId = 0 AND Bluuid IS NOT NULL

CREATE UNIQUE INDEX IX_UniqueDeviceId on Trace_Product (DeviceId, ProductRecordStateId) WHERE ProductRecordStateId = 0

CREATE UNIQUE INDEX IX_UniqueSerialNo on Trace_Product (SerialNo, ProductRecordStateId) WHERE ProductRecordStateId IN(0,2)
